### 1. Ejemplo para OCI Data Flow (usando Spark)


Como te mencion√©, el script actividades/spark/main.py es un excelente punto de partida. Para la clase, puedes usarlo para explicar los conceptos b√°sicos de Spark.

C√≥digo de `actividades/spark/main.py`:

```python
from pyspark.sql import SparkSession
from pyspark.sql.functions import col

# 1. Crear la sesi√≥n de Spark
spark = SparkSession.builder \
    .appName("Ejemplo PySpark") \
    .getOrCreate()

# 2. Crear un DataFrame (en un caso real, leer√≠amos de OCI Object Storage)
data = [
    {"nombre": "Rodrigo", "edad": 34, "ciudad": "Santiago"},
    {"nombre": "Juan", "edad": 29, "ciudad": "Valpara√≠so"},
    {"nombre": "Camila", "edad": 40, "ciudad": "Concepci√≥n"}
]
df = spark.createDataFrame(data)

print("üìã DataFrame original:")
df.show()

# 3. Filtrar personas mayores de 30
df_filtrado = df.filter(col("edad") > 30)

print("üë§ Personas mayores de 30:")
df_filtrado.show()

# 4. Calcular edad promedio
df.groupBy().avg("edad").show()

# 5. Ejecutar una consulta SQL
df.createOrReplaceTempView("personas")
resultado_sql = spark.sql("SELECT ciudad, COUNT(*) as cantidad FROM personas GROUP BY ciudad")
print("üèôÔ∏è Cantidad de personas por ciudad:")
resultado_sql.show()
```

Explicaci√≥n para la clase: Este script se puede ejecutar en OCI Data Flow para procesar terabytes de datos. El √∫nico cambio ser√≠a reemplazar el paso 2 para leer los datos desde 
un archivo en un bucket de OCI.

### 2. Ejemplo para OCI AI Services (Language)

Aqu√≠ tienes un script de Python para realizar an√°lisis de sentimiento con el servicio OCI Language.

Prerrequisitos:
*   Instalar el SDK de OCI: pip install oci
*   Tener configuradas las credenciales de OCI en tu m√°quina.

C√≥digo de ejemplo:

```python
# Este script requiere el SDK de OCI y configuraci√≥n previa.
import oci
from oci.ai_language.ai_service_language_client import AIServiceLanguageClient
from oci.ai_language.models.batch_detect_language_sentiments_details import BatchDetectLanguageSentimentsDetails
from oci.ai_language.models.text_document import TextDocument

# --- Configuraci√≥n ---
# Carga la configuraci√≥n desde el archivo por defecto (~/.oci/config)
config = oci.config.from_file()
# IMPORTANTE: Reemplaza esto con el OCID de tu Compartment
compartment_id = "ocid1.compartment.oc1..xxxxxxxxxxxxxxxxxxxxxxxxxx" 

# Textos para analizar
documentos = [
    TextDocument(key="doc1", text="El servicio al cliente fue excelente, resolvieron mi problema r√°pidamente.", language_code="es"),
    TextDocument(key="doc2", text="Estoy muy decepcionado con la calidad del producto, no lo recomiendo.", language_code="es"),
    TextDocument(key="doc3", text="El paquete lleg√≥ a tiempo, pero el embalaje estaba da√±ado.", language_code="es")
]

# --- Cliente y llamada a la API ---
language_client = AIServiceLanguageClient(config)
batch_details = BatchDetectLanguageSentimentsDetails(documents=documentos)

try:
    response = language_client.batch_detect_language_sentiments(
        batch_detect_language_sentiments_details=batch_details,
        compartment_id=compartment_id
    )

    # --- Resultados ---
    print("Resultados del An√°lisis de Sentimiento:")
    for doc in response.data.documents:
        aspect = doc.aspects[0] # Simplificamos al primer aspecto detectado
        print(f"- Documento '{doc.key}':")
        print(f"  - Sentimiento: {aspect.sentiment}")
        print(f"  - Scores -> Positivo: {aspect.scores.positive:.2f}, Negativo: {aspect.scores.negative:.2f}, Neutral: {aspect.scores.neutral:.2f}")

except oci.exceptions.ServiceError as e:
    print(f"Error al llamar al servicio de OCI Language: {e.status} {e.message}")
```

Explicaci√≥n para la clase: Este c√≥digo muestra lo f√°cil que es integrar un modelo de IA complejo (an√°lisis de sentimiento) en una aplicaci√≥n. Simplemente se env√≠an los textos a
la API de OCI y se obtiene un resultado estructurado (positivo, negativo, neutral) sin haber entrenado ning√∫n modelo.
